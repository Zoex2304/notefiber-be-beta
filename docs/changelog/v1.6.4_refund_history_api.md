# v1.6.4 - Refund History API

## Overview
Following the Cancellation History update, this release enables users to track their Refund requests with improved visibility.

## New Features

### Refund Request History

#### List Refunds
*   **Endpoint**: `GET /api/user/refunds`
*   **Description**: Lists all refund requests submitted by the user.
*   **Response**:
    ```json
    {
        "success": true,
        "message": "User refunds",
        "data": [
            {
                "id": "uuid",
                "subscription_id": "uuid",
                "plan_name": "Pro Plan",
                "amount": 50000,
                "reason": "Duplicate charge",
                "status": "pending",
                "created_at": "2026-01-04T12:00:00Z"
            }
        ]
    }
    ```

#### Refund Detail
*   **Endpoint**: `GET /api/user/refunds/:refundId`
*   **Description**: Retrieves detailed info for a specific refund request.
*   **Response**: Same structure as list item.
*   **Errors**: `404 Not Found` if ID invalid/unauthorized.

## Improvements
*   **Consistency**: Aligned Refund API structure with Cancellation API for uniform frontend integration.
